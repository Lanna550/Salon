# AGENTS.md

# Mission

Build a luxury Korean beauty salon website.

The website must feel like a real premium beauty brand operating in Seoul, Gangnam, or a luxury Australian Korean salon.

Every implementation decision must prioritize elegance, trust, femininity, and professionalism.

The final experience should feel handcrafted by an experienced design team.

Never generate layouts that resemble AI-generated SaaS templates.

---

# Core Philosophy

Luxury is created through restraint.

Not through:

- flashy effects
- excessive animation
- gradients
- oversized UI elements
- visual clutter

Luxury is created through:

- spacing
- typography
- photography
- consistency
- hierarchy

---

# Absolute Design Rules

The website MUST NOT resemble:

- startup landing pages
- software company websites
- crypto websites
- AI product pages
- dashboards
- admin templates

If a section looks like a SaaS homepage, redesign it.

---

# Visual Priority

When making design decisions:

1. Photography
2. Typography
3. Layout
4. Spacing
5. Color
6. Animation

Animation is always the lowest priority.

---

# User Journey

The user should experience:

Arrival

↓

Luxury Impression

↓

Trust Building

↓

Service Discovery

↓

Social Proof

↓

Team Confidence

↓

Booking Intent

↓

Conversion

Every page must support this flow.

---

# Layout Philosophy

Use generous whitespace.

Sections should never feel compressed.

Each section should feel intentionally curated.

Do not stack too much information vertically.

Maintain visual breathing room.

---

# Component Philosophy

Every component must answer:

"Does this increase trust?"

If not:

remove it.

---

# Development Constraints

Before creating any UI component:

Ask:

Would this exist on a luxury salon website?

If uncertain:

do not implement.

---

# Motion Philosophy

Motion supports content.

Motion never becomes content.

Animations must be nearly invisible.

If animation becomes noticeable:

it is too strong.

---

# Code Philosophy

Prioritize:

- maintainability
- readability
- accessibility
- semantic structure
- performance

Never sacrifice usability for visual effects.
